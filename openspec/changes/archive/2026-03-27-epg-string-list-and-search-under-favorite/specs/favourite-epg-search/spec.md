## ADDED Requirements

### Requirement: EPG 频道搜索页

系统在侧栏「想看的频道」分组下 SHALL 提供名为「EPG 频道搜索」的菜单入口及对应页面。用户在该页面输入频道标识并发起查询时，系统 MUST 使用 `GET /epg?channel=<用户输入>` 获取数据，并将返回的节目信息以可读形式展示；失败或空数据时 MUST 给出明确提示且不崩溃。

#### Scenario: 查询成功

- **WHEN** 用户输入非空频道标识并触发查询且接口返回成功
- **THEN** 系统在页面中展示返回的节目信息

#### Scenario: 查询失败或空结果

- **WHEN** 接口错误或返回空数据
- **THEN** 系统展示错误信息或「暂无数据」类提示

#### Scenario: 菜单位置

- **WHEN** 用户展开侧栏「想看的频道」
- **THEN** 可见「EPG 频道搜索」子菜单项，且与「想看的频道」主内容入口并列展示（具体子项集合以实现阶段路由为准，但至少包含本入口）
