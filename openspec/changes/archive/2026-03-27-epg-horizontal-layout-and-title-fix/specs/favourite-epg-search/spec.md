## MODIFIED Requirements

### Requirement: EPG 频道搜索页

系统在侧栏 SHALL 提供名为「EPG 频道搜索」的**顶层**菜单入口（与「想看的频道」**同级**，非其子菜单），并对应独立路由页面（路径以实现为准，如 `/epg-channel-search`）。旧路径（如 `/favorite/epg-search`）MAY 重定向至新路径。用户在该页面输入频道标识并发起查询时，系统 MUST 使用 `GET /epg?channel=<用户输入>` 获取数据，并将返回的节目信息以可读形式展示；失败或空数据时 MUST 给出明确提示且不崩溃。
展示的节目信息 MUST 按照日期分组，日期 MUST 以横向（如 Tab 页）形式展示，用户可切换日期。对于选中的日期，每天的节目从 0 点开始顺序竖排展示，并且 MUST 用粗体绿色文字高亮展示当前正在播放的节目信息。同时，系统 MUST 正确解析节目名称（支持从 `titles[0].value` 提取）。

#### Scenario: 查询成功

- **WHEN** 用户输入非空频道标识并触发查询且接口返回成功
- **THEN** 系统在页面中展示返回的节目信息，日期横向排列供选择，选中日期的节目从 0 点开始顺序竖排展示，正确显示节目名称，并用粗体绿色文字高亮当前正在播放的节目

#### Scenario: 查询失败或空结果

- **WHEN** 接口错误或返回空数据
- **THEN** 系统展示错误信息或「暂无数据」类提示

#### Scenario: 菜单位置

- **WHEN** 用户查看侧栏中与「想看的频道」相同层级的菜单列表
- **THEN** 可见「EPG 频道搜索」为独立项，且点击后进入上述页面
